Java-Web-JSP EL JSTL

Posted on 2018-05-22 | In JAVA 基础学习笔记 , Java Web
Words count in article: 1,632

JSP

JSP,即 Java Server Pages。Java服务器页面，运行在服务端的页面。后缀名为 .jsp 。JSP文件本质是Servl，不同的是JSP用于进行数据展示的Servl。普通的Servlet是用于完成业务逻辑处理。结合html。

JSP中的Java代码块

JSP中的Java代码块也称为JSP小脚本，用 <% %> 括起来的部分。

  1. <% %> 不能加权限访问 private，不能定义方法，不能定义静态语句块
  2. <%! %> 可以加权限访问，可以定义方法，可以定义静态语句块
  3. <%= %> 表达式块，将数据显示到客户端浏览器
  4. <%@ page import=” “ %> 导入包，多个包用 ， 隔开

JSP指令

作用：为当前页面做一些基本的属性设置。

  1. page: 页面指令
  2. include: 包含指令
  3. taglib: 标签库指令

格式： <%@ 指令名称 属性名=属性值 ……%>

page指令

一个JSP文件可包含多个page指令，设置当前页面的相关信息。

  1. pageEncoding 用于设置当前页面的的字符编码格式
  2. import 在JSP中导入指定的类，多个类用 ， 隔开
  3. contentType 设置JSP页面呈现于用户流浪器中的内容类型 “text/html”
  4. errorPage 当运行发生异常时跳转的页面 “/error.jsp”
  5. isErrorPage 当一个页面的 page 指令中设置 isErrorPage 的值为 true 时，表明当前页面为一个“错误处理页面”。默认 isErrorPage 的值为 false
  6. session 判断是否可以直接使用内置对象session，默认是 true

    include指令

    该指令只有一个属性 file ，指定要包含的文件。为了减少页面定义的重复性工作。
    <%@ include file=”/***.jsp” %>

JSP动作

JSP动作： 使用系统定义好的标签来完成本应由Java代码来完成的功能。
<jsp:动作名称 属性名=属性值 ……></jsp:动作名称>
<jsp:动作名称 属性名=属性值 ……/>

forward动作

从当前JSP转到另一个JSP，servlet或者Web应用中的任何其他资源，

inlcude动作

被包含的页面要出现在发送请求的页面。比如一个页面的页眉，它会出现在每一个页面，使用include动作可以实现。
不同于include指令，include指令在运行时发生，jsp:include动作在运行时发生。include指令的属性是file，inlcude动作的属性是page.

EL

Expression Lang，表达式语言。在JSP页面任何静态部分可用 ${expression}获取指定表达式的值。只能从四大域中获取数据。

  1. pageContext pageScope （从指定域中查找对象）
  2. request requestScope
  3. session sessionScope
  4. application applicationScope

如果访问对象为 null, EL不会抛出空指针异常，会显示为空。

内置对象

  1. param 获取请求参数指定参数值
  2. paramValues 获取请求参数的所有值
  3. initParam 获取初始化参数
  4. pageContext 获取路径访问路径 ${ pageContext.request.contextPath }/

自定义函数

因为EL不支持字符串的操作，使用自定义函数完成。

定义一个静态方法类

package com;
public class ELFuntions {
public static String LowerToUpper(String s)
    {
        return s.toUpperCase();
    }
}

注册函数，配置 .tld

注册函数，在 Web 项目的/WEB-INF 目录下，新建一个扩展名为.tld 的 XML 文件，例如 myfn.tld。tld，即 Tag Library Definition，标签库定义。将定义好的函数，在这个文件中进行注册。XML文件需要约束，需要配置文件头暗转Tomcat安装目录下：tomcat-9\webapps\examples\WEB-INF\jsp2 找到 jsp2-example-taglib.tld
在 .tld 中添加

<taglib xmlns="http://java.sun.com/xml/ns/j2ee"
xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xsi:schemaLocation="http://java.sun.com/xml/ns/j2ee http://java.sun.com/xml/ns/j2ee/web-jsptaglibrary_2_0.xsd"
version="2.0">

//添加子标签
<tlib-version>1.0</tlib-version>
<short-name>test</short-name>    //最好与文件名相同,在JSP页面中要使用
<uri>http://www.carloshaoxj.cn/jsp/el/funtions</uri>//指定该函数所对应的URI

//注册函数
<function>
    <name>LowerToUpper</name>    //与指定函数的名称相同
    <function-class>com.ELFuntions</function-class> //类的名称
    <function-signature>java.lang.String LowerToUpper( java.lang.String )</function-signature>
</function>

</taglib>

使用自定义函数

<%@ taglib uri="http://www.carloshaoxj.cn/jsp/el/funtions" prefix="text"%>
...
<body>
<%
    request.setAttribute("name","zhangsan");    //必须存入四大域中
%>
${text:LowerToUpper(name)}
</body>
...

定制标记

使用EL和标准动作是有限制的，有些功能不可完成，比如数组处理功能。所以引入了定制标记来完成不可完成的功能。

安装 JSTL

使用 JSTL 之前需要将 jstl.jar 和 stadard.jar 导入到 Web 应用的 WEB-INF/lib 目录下。

JSTL 标记语法

在 jsp 头部加入 <%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %> 使用 JSTL

  1. 核心标记：out, set, remove, catch
  2. 条件标记：if, choose, when, otherwise
  3. 循环标记：forEach,
  4. URL标记：url

核心标记

c:out

用来输出表达结果，使用 <c:out value="string" default="string" escapeXml="true | false"/c:out>

  1. value: 要输出的内容
  2. default: 输出默认值
  3. escapeXml: 是否忽略XML特殊字符 默认 true

c:set

  • 用来设置变量值和对象属性。假设存在一个集合 Map ，想在该集合创建新的内容使用 <c:set> 来完成。
  • <c:set var="user" scope="string" value="string"/>
  • <c:set target="string" propety="string" value="string"/>
  1. var:如果有一个名为 user 的域属性，创建一个的属性，value 不能为空
  2. value:要存储的值，如果为空，则会删除该变量
  3. scope:var属性作用域
  4. proprty:要修改的属性
  5. target:要修改的属性所属的对象
  • 不能同时拥有 var 和 target

    c:remove

    用于移除一个变量
  • <c:remove var="" scope=""/>
  1. var:要移除的变量名称，必须是一个String直接量，不能是表达式
  2. scope:变量所属作用域

    c:catch

    处理产生错误的异常状况，并且将错误信息储存起来
  • <c:catch> var=""> ... </c:catch>
  • var:储存错误信息的变量

条件标记

c:if

判断表达式值

  • <c:if test="boolean" var="string" scope=""> ... </c:if>
  1. test:条件
  2. var:用来存储条件结果的变量
  3. scope:var属性的作用域

    c:choose

    与 switch 功能一样。
    <c:choose>
    <c:when test="boolean">
    ...
</c:when>
<c:otherwise>
    ...
</c:otherwise>
    </c:choose>
  • test:条件判断

    c:when

    c:choose 的子标签，判断是否成立

    c:otherwise

    c:choose 的子标签，接在 c:when 标签后。当 c:when 判断为 false 时执行

循环标记

c:foreach

迭代集合对象

  • <c:foreach var="" items="">
  1. var:当前变量名称
  2. items：要被循环的信息

    URL标记

    URL 格式化为一个字符串，
  • <c:url var="" value="">
  1. var:URL变量名
  2. value:路径
